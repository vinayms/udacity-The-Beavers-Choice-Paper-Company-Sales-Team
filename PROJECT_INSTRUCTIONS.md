# Project Instructions: The Beaver's Choice Paper Company Sales Team

## Step 1: Draft Your Agent Workflow
Begin by drafting a diagram illustrating the interactions and data flows between the agents in your multi-agent system. Your diagram should demonstrate the sequence of operations for handling customer inquiries, inventory management, quote generation, and order fulfillment. 

### Recommended Agents:
* **Orchestrator Agent**: Handles customer inquiries and delegates tasks to specialized agents.
* **Inventory Agent**: Answers inventory queries accurately, including deciding when to reorder supplies.
* **Quoting Agent**: Generates quotes efficiently, applying bulk discounts strategically to encourage sales.
* **Sales Agent**: Finalizes sales transactions, considering inventory levels and delivery timelines.

These agents should have access to tools that interact with the system database and the outside world.

### Recommended Tools:
* **Check Inventory Tool**: Checks inventory for different paper types.
* **Get Quote History Tool**: Gets quote history related to a customer's request.
* **Check Supplier Delivery Timeline Tool**: Checks the timeline for delivery of an item from the supplier.
* **Fulfill Order Tool**: Fulfills orders by updating the system database.

You can use diagramming tools like Diagrams.net or Mermaid to create your diagrams.

---

## Step 2: Review the Starter Code
Carefully examine the provided starter code (`project_starter.py`) in your workspace. This code includes essential functionalities such as:
* Initializing and managing an SQLite database
* Managing inventory stock levels
* Generating and tracking financial transactions
* Utility functions to estimate supplier delivery dates and current cash balance

At the bottom of the `project_starter.py` file, you'll find a provided code stub designed to help you evaluate your agent implementation. You can use this stub to test and refine your system effectively.

*Spend at least 30 minutes reviewing the starter file. Write brief descriptions for each function provided to ensure you thoroughly understand their purpose and usage within your system.*

Once you complete your review, update the tools you initially outlined, replacing hypothetical tools with tools defined using the helper functions provided in the starter code.

---

## Step 3: Select Your Agent Framework
Decide on the agent orchestration framework you will use for this project. Your options include:
* `smolagents`
* `pydantic-ai`
* `npcsh`

---

## Step 4: Implement Your System
Using the framework you have selected, implement the agents based on the updated flow. Ensure your system follows the agent workflow diagram. 

Create specialized agents for each task, and an orchestration agent that orchestrates the flow of control and data between them. Use the helper functions provided in the starter file to define tools for the worker agents.

---

## Step 5: Test and Evaluate Your Implementation
Test your multi-agent system thoroughly using the provided dataset (`quote_requests_sample.csv`). Ensure:
* Your agents correctly handle various customer inquiries and orders.
* Orders are accommodated effectively to optimize inventory use and profitability.
* The quoting agent consistently provides competitive and attractive pricing.

Refer to the project rubric to evaluate the results documented in the `test_results.csv` file.

---

## Step 6: Reflect and Document
After evaluating your system, prepare a clear and concise report detailing:
* A comprehensive explanation of your multi-agent system.
* Evaluation results (`test_results.csv` generated by the evaluation code) highlighting strengths and areas for improvement.
* Suggestions for further improvements based on the areas of improvement.

### Your final submission should include:
1. Your updated agent flow diagram (image file).
2. Your completed implementation script.
3. Your reflective report with evaluations.
